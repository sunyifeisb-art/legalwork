# 数据合规AI项目母包

这是给团队内部继续研发、协作、测试、分发用的**项目骨架**。

## 当前定位
- 中国法语境下的数据合规 AI 审查工作流引擎
- 第一阶段聚焦：**对数据合规相关文件或文本做系统性风险审查**
- 产品定位：**辅助合规/法务提高效率**，不替代律师或法务作最终判断

## 第一阶段唯一闭环
输入一份文件或一段文本，系统尽可能通过多种方式结合外部规范数据库完成审查，输出：
- 风险点
- 法规依据
- 风险等级
- 修改建议
- 是否触发自动复核
- 自动复核统计与待补支持信息
- 同主题风险聚类与交叉印证结果
- 自动生成整改任务清单（按 P1/P2/P3 排序）
- 整改前后自动回归验证（比较风险是否下降）
- 系统性报告

## 目录说明
- `docs/`：项目说明、范围、风险台账、应用层场景包说明
- `skill/`：项目专用 skill 及引用说明
- `workflows/`：三条主工作流骨架
- `knowledge-base/`：外部规范数据库对接时使用的占位规范（当前不作为主任务）
- `samples/`：样本输入与评测样本位
- `templates/`：固定输出模板
- `tests/`：第一阶段验收与评测规则
- `config/`：项目配置模板、结构 schema、应用层场景、证据要求、SDK专项包、数据出境专项包、隐私整改专项包、路由/检查点/方法/优先级/外部适配配置
- `scripts/`：初始化、预处理、识别、路由、任务生成、规则化审查、结果汇总、外部映射、应用层场景规划、证据链清单生成、SDK专项包生成、数据出境专项包生成、隐私整改专项包生成、报告生成、总编排、精度回归、smoke check 脚本

## 当前不纳入第一阶段
- OCR
- 扫描件/图片复杂解析
- 多模态复杂恢复
- 数据库建设与法规整理体系
- 完整法规自动监控系统

## 第一阶段默认验收重点
1. 检索是否稳
2. 判断是否稳
3. 报告是否可读
4. 平台是否扛得住长文本和批量材料

## 使用顺序
1. 先看 `docs/first-phase-scope.md`
2. 如需对接外部规范数据库，再参考 `knowledge-base/` 占位规范
3. 用 `workflows/02-policy-review/` 跑第一条主链路（现已定义为通用合规文件审查）
4. 用 `tests/eval-metrics.md` 做评估
5. 跑 `scripts/run_precision_regression.py` 看误报/漏报回归是否稳定
6. 把结论同步回项目笔记与迭代记录

## 数据源约束（新增）
- 后续该项目的训练样例、规则校准、法规映射、评测材料，默认优先使用**数据合规知识库**中的真实数据/真实材料。
- 在知识库接入完成前，可以继续搭工作流骨架与解析逻辑，但**不得把我临时编造的数据当作正式训练/校准数据**。
- 如果当前环境拿不到知识库访问入口、导出文件或同步接口，必须明确记录为阻塞项。
- 当前已定位到两个 IMA 知识库：`数据合规知识库` 与 `网络安全与数据合规`（见 `config/ima-knowledge-base.json`），它们在当前阶段仅作为**训练/迭代数据库**使用。
- 但在当前凭证下读取这两个知识库的目录/内容仍被 IMA API 拒绝（`220030 / 没有权限`），因此现在只能确认知识库存在，不能假装已把其内容喂进工作流。

## 当前应用层推进（新增）
- 当前阶段不先做管理后台，而是把应用层先收成几个场景包：
  - 产品合规测评
  - 隐私文档整改
  - 数据出境专项审查
  - SDK与合作方合规审查
- 主流水线在输出风险报告后，会继续生成一份 `application scenario plan`，把规则层和技术层结果翻译成场景化交付清单。
- 同时继续生成一份 `evidence checklist`，把每个风险点需要补的合同、截图、授权记录、制度材料等证据链拆出来，便于后续自动复核与整改留痕。
- 对命中 `SDK与合作方合规审查` 场景的材料，还会继续生成 `sdk partner review pack`，把第三方清单、对外披露对齐、合同检查、整改动作收成专项包。
- 对命中 `数据出境专项审查` 场景的材料，还会继续生成 `cross border review pack`，把境外接收方、出境场景与必要性、保障机制、整改动作收成专项包。
- 对命中 `隐私文档整改` 场景的材料，还会继续生成 `privacy remediation pack`，把条款缺口、补写优先级、第三方口径整改、重写输出收成专项包。

## 当前规则引擎口径（本轮增强）
- 不再只靠“命中关键词就报风险”
- 改为：**触发主题 + 关键要素覆盖度 + 模糊表述识别 + 场景化升级规则**
- 对于披露已经较完整的文本，默认压低误报，不强行报中风险
- 对于“营销/推荐却缺少同意基础”“长期保存却没有删除安排”等场景，允许直接升级为高风险
- 默认会自动套用一层**法规映射**，把核心风险项落到具体法条，而不再只显示占位依据
- 新增五条核心审查路径：**目的匹配与过度收集检查**、**敏感个人信息专项检查**、**字段—目的—法律基础三联校验**、**同意与可选功能绑定检查**、**前后表述一致性检查**
