互联网域名服务信息安全管理要求
1范围

本文件规定了互联网域名服务在信息安全方面的管理要求。
本文件适用于在中华人民共和国境内从事互联网域名服务的相关机构，包括域名根服务器（含镜像
服务器）运行、域名注册管理、域名注册服务、域名权威解析和域名递归解析等。

2规范性引用文件

本文件没有规范性引用文件。

3术语和定义

下列术语和定义适用于本文件。

3.1

域名 domain name

互联网络上识别和定位计算机的层次结构式的字符标识，与该计算机的互联网协议 （ IP） 地址相对
应。

3.2

顶级域名 top level domain name

域名体系中根节点下的第一级域的名称。

3.3

域名根服务器 root server of domain name

承担域名体系中根节点功能的服务器（含镜像服务器）。

3.4

域名根服务器（含镜像服务器）运行机构 operator of domain name root server including the mirror
server

依法获得许可，承担域名根服务器（含镜像服务器）运行、维护和管理工作的机构。

3.5

域名注册管理机构domain name registry
承担顶级域名系统的运行、维护和管理工作的机构。

域名注册服务机构domain name registration
受理域名注册申请，直接完成域名在国内顶级域名数据库中注册，直接或间接完成域名在国外顶级
域名数据库中注册的机构。

3.7

域名注册代理机构 domain name Regustration Agent）
受域名注册服务机构的委托，受理域名注册申请，间接完成域名在顶级域名数据库中注册的机构。

3.8

域名解析服务 domain name resolution
在特定区域内接收用户对有关域名和资源记录信息查询，响应请求结果的解析服务，域名解析通常
包括域名权威解析和递归解析两类。

3.9

域名权威解析服务 domain name authoritive resolution
在某个或者多个区域（即权威域）内，提供原始（权威）域名资源记录信息的解析服务。

3.10

域名递归解析服务 domain name recursive resolution
接受用户的解析请求，通过向权威服务器发出查询请求获得用户所需资源记录信息，并最终向用户
返回结果的解析服务。

3.11

域名月艮务机构 domain name service organization

向公众用户提供域名根服务器（含镜像服务器）运行、域名注册管理、域名注册服务（含代理）、
域名托管/权威解析、域名递归解析等服务的组织机构。

3.12

违法违规域名illegal domain name

含有《互联网域名管理办法》第二十八条第一款所列内容的域名，以及电信管理机构认定的存在违
法违规行为的域名。

5概述

本文件根据域名服务机构开展的域名服务类型（域名根服务器运行、域名注册管理、域名注册服务、
域名解析服务），以及域名服务机构的信息安全责任，对域名根服务器运行机构、域名注册管理机构、
域名注册服务机构和提供域名解析服务的组织机构在信息安全管理机构、信息安全人员配备、信息安全
管理制度、以及应急处理等方面分别提出了管理要求。

6域名根服务器（含镜像服务器）运行机构信息安全管理要求

6. 1信息安全管理机构

6.1.1 部门及责任人设置

域名根服务器（含镜像服务器）运行机构应健全内部信息安全管理体系，设置或指定本企业的信息
安全主管部门，负责本企业信息安全相关工作。
域名根服务器（含镜像服务器）运行机构法定代表人为本单位信息安全第一责任人，对企业内的信
息安全工作负有领导责任；企业分管信息安全的负责人为本单位信息安全责任人，对本企业信息安全工
作负直接领导责任。

6. 1.2岗位设置

域名根服务器（含镜像服务器）运行机构应设立信息安全主管负责人岗位、信息安全管理员岗位，
定义工作岗位的职责并规定岗位从业资质。

6. 1.3工作职责

域名根服务器（含镜像服务器）运行机构的信息安全主管部门应制定本企业信息安全工作总体目标
和总体信息安全策略，应建立健全企业信息安全责任制度，制定本企业信息安全相关管理规范、技术标
准和预案，明确企业各部门信息安全职责及工作要求。企业信息安全主管部门的工作职责至少包括：
a） 督促企业内部履行信息安全职责，指导企业内其他部门开展信息安全相关工作；
b） 负责企业内部信息安全检查，并对企业内部信息安全管理工作进行监督、考核；
c） 负责建设与业务规模配套的信息安全保障措施和技术手段；
d） 负责企业各类信息安全事件的响应、监控、处置等工作，并同时向电信管理机构报告；
e） 负责企业内部信息安全相关教育、培训工作；
f） 负责企业内部信息安全相关应急预案的定期培训、演练等。
6. 2信息安全人员配备

6.2.1 人员数量

域名根服务器（含镜像服务器）运行机构应配备至少2名专职信息安全管理员和1名信息安全应急
联系人，应急联系人需提供中国境内的通信联系方式，如信息安全应急联系人或者联系方式有变动的，
应自变动之日起两个工作日内主动向电信管理机构书面报告。

6. 2. 2人员资质

承担域名根服务器（含镜像服务器）运行机构信息安全管理工作的人员应熟悉行业信息安全管理相
关法律法规和政策要求。

6. 3.1管理制度

域名根服务器（含镜像服务器）运行机构应制定并完善信息安全管理制度，明确信息安全工作的目
标、范围、方针、策略、原则、责任、实施方案、应急预案等，具体包括：
a） 信息安全责任考核和奖惩制度；
b） 重大信息安全事件应急处置和报告制度；
c） 信息安全教育、培训、演练制度；
d） 保障企业落实信息安全管理责任的其他制度。

6. 3. 2 制定和发布

域名根服务器（含镜像服务器）运行机构的信息安全主管部门负责信息安全管理制度的制定，并对
相应的文件进行管理。
信息安全管理制度应具有统一的格式，并应进行版本控制。有密级的管理制度应注明密级，并按相
应密级进行管理。信息安全主管部门应组织相关部门和相关人员对制定的信息安全管理制度进行论证和
审定。在经过审定，并通过领导或相关人员审批后，由行政部门或信息安全主管部门进行发布，在发布
时应注明发布范围，并对发文进行登记。

6. 3. 3评审和修订

域名根服务器（含镜像服务器）运行机构的信息安全主管部门应根据信息安全管理制度的相应密级
确定评审和修订的操作范围。信息安全主管部门定期组织相关部门和相关人员对信息安全管理制度体系
的合理性和适用性进行评审，并定期或不定期对安全管理制度进行评审。在发现管理制度和制度体系中
存在不足和需要改进的条款后，信息安全主管部门应即时组织相关部门和人员进行修订。
6. 4应急处理

6. 4.1安全事件处理

域名根服务器（含镜像服务器）运行机构应具备以下安全事件处理能力：
a） 域名根服务器（含镜像服务器）运行机构应对信息安全事件进行明确分级（特别重大事件、
重大事件、较大事件和一般事件）no
b） 对不同级别的信息安全事件，域名根服务器（含镜像服务器）运行机构应制定相应的应急
处理预案。
c） 域名根服务器（含镜像服务器）运行机构应制定安全事件报告和处置管理制度，规定安全
事件的现场处理、事件报告和后期恢复的管理职责。
d） 域名根服务器（含镜像服务器）运行机构应记录并保存所有发现和处理的信息安全事件，
分析并记录事件原因，处理措施，总结经验教训。
e） 域名根服务器（含镜像服务器）运行机构应指定应急联系人，联系人应保持7*24小时联系
渠道畅通。
f） 域名根服务器（含镜像服务器）运行机构应制定7*24值班制。

6. 4.2应急处理预案

域名根服务器（含镜像服务器）运行机构应具备以下应急预案管理措施：
a） 应急预案应确定处理信息安全事件时所要采取的具体步骤、确定发生情况的应对处置措施。
b） 应有专门人员负责在启动应急预案时与其他相关单位（如网络运营商等）的协调工作。

7. 5业务技术合作要求

7. 5.1服务协议

域名注册管理机构在与第三方域名技术服务提供者开展业务技术合作前，应对合作方的经营资质、
业务许可、信息安全技术保障能力等进行审核。
域名注册管理机构在与第三方域名技术服务提供者开展业务技术合作时，应统筹考虑信息安全管理
需要，在双方服务协议中明确信息安全责任分工，明确第三方域名技术服务提供者有配合信息安全保障
工作的义务，明确其有责任按照电信管理机构要求落实网络信息安全相关工作，在涉及国家安全或处置
紧急事件需求时，应服从电信管理机构的统一指挥和协调，遵守并执行电信管理机构的管理要求。

7.5.2日常管理

域名注册管理机构应要求第三方域名技术服务提供者定期向其汇报域名系统运行状况及信息安全
工作落实情况，并监督第三方域名技术服务提供者落实信息安全日常管理工作。
7. 6违法违规域名管理

7. 6.1违法违规域名发现

域名注册管理机构应定期对二级域名展开审查工作，对于存在违法违规操作的二级域名，经核实后
及时处置，情节特别严重的应上报电信管理机构。

7. 6.2违法违规域名处置

域名注册管理机构应建立违法违规域名处置机制。按照违法违规情节严重性可以分为一般、严重和
特别严重。
对于一般违法违规域名，域名注册管理机构应向域名持有人下发违法违规域名处置通知，要求限期
整改；对于严重违法违规域名或在规定时间内未完成整改的一般违法违规域名，域名注册管理机构应停
止该域名的解析；对于特别严重违法违规域名，域名注册管理机构应注销该域名，停止对其的所有域名
服务。
域名注册管理机构应配合电信管理机构进行违法违规域名处置工作，停止违法违规域名的权威解析
或注销违法违规域名，并确保7*24小时均能及时响应。

7. 6.3违法违规域名列表联动

域名注册管理机构应与有合作关系的域名注册服务机构建立违法违规域名联动机制，及时与有合作
