# Skill: Compare DDRL to VDR Index (Scenario 02)

## 1. Subject-matter triage

- Treat the DDRL, VDR index, status tracker, and associate notes as a single diligence record set; reconcile them before drafting any conclusions.
- If the source set contains multiple business lines, entities, periods, or diligence workstreams, enumerate them first and run the gap analysis separately for each before rolling up themes.
- If only one entity, period, or workstream is actually in scope, say so expressly and do not imply broader coverage.

## 2. Failure modes the skill is correcting

- Listing missing items without classifying whether the issue is a simple omission, a compliance deficiency, or a substantive ownership / chain-of-title defect.
- Treating all gaps as equal and failing to identify which ones merit pre-closing remediation, escalation, or continued tracking.
- Ignoring status nuances in the source set, including items that are partially satisfied, deferred, resisted, or pending follow-up.
- Missing technology, privacy, cybersecurity, operational, or other non-core diligence categories when they appear in the request set.
- Describing issues without linking them to the affected source document set, the consequence for the transaction, and the next step required to close the gap.
- Failing to surface the legal or regulatory authority that makes a missing item material when the issue implicates compliance, approval, filing, permit, or similar obligations.

## 3. Legal frameworks / domain conventions that apply

- A gap analysis memo should distinguish document production status from substantive diligence risk: a missing artifact may be only an information gap, but it may also evidence a live compliance problem or a defect in title, authority, or entitlement.
- Where the missing item concerns filings, renewals, registrations, approvals, or permits, assess the issue against the governing statute, regulation, or approval regime reflected in the source materials; identify whether the problem is timing, omission, or non-compliance.
- Where the missing item concerns ownership of assets, inventions, source code, works product, real property, or other transferred rights, assess whether the absence of a transfer, assignment, or similar instrument points to a chain-of-title issue requiring pre-closing cure.
- Where the source set addresses restrictive covenants, retention protections, or key personnel arrangements, assess the relevance of any missing documents in light of the transaction context and the personnel’s role.
- Where the diligence request covers technology, privacy, or cybersecurity, analyze those materials with the same rigor as legal and financial diligence categories; incomplete security, privacy, or incident-response materials may indicate regulatory exposure as well as diligence delay.
- Use controlling authority when stating a legal proposition; cite the statute, regulation, rule, or other authority that supports the conclusion whenever the source set identifies it or the governing authority is otherwise clear.

## 4. Analytical scaffolds

- Start with a short executive summary that states the overall diligence posture and identifies the most material gaps.
- Define one ordinal severity scale at the outset and apply it uniformly to every issue entry; keep the scale simple and consistent across the memo.
- For each requested item or issue, capture:
  - the request reference or category;
  - the VDR / tracker / notes status;
  - the gap characterization;
  - the legal or transactional significance;
  - the recommended follow-up.
- For each substantive gap, close the analysis by:
  - tying the gap to a concrete source-document measure, date, status, or other relevant anchor;
  - cross-referencing any related request, tracker entry, or note that interacts with the issue;
  - stating the downstream consequence for the deal, compliance posture, or closing condition.
- Separate pure document omissions from items suggesting a compliance deficiency, and separate both from items suggesting a title, authority, or entitlement defect.
- If the source materials show conflicting statuses, reconcile them explicitly rather than choosing one source silently; explain which source controls and why.
- If a document is partially provided, describe what is covered and what remains missing; do not collapse partial production into full satisfaction.
- When the analysis spans several categories, use the same evaluative method in each category so the reader can compare gaps across workstreams.
- End every issue entry with a concrete action request, the role that should own it, and a timing anchor tied to the diligence or transaction timetable.

## 5. Vertical / structural / temporal relationships

- Track how the DDRL request, VDR production status, and associate notes relate to one another over time; a later tracker update may supersede an earlier index entry.
- If timing matters, distinguish items that were due by the diligence cutoff from items that are merely outstanding as of the review date.
- Flag gaps that appear to worsen over time, such as repeated deferrals, unanswered follow-ups, or inconsistent responses across versions of the source set.
- Where a missing item is linked to a pre-closing covenant, closing condition, or post-signing deliverable, state that relationship clearly.
- If an item is available only through a substitute document, explain the substitute’s limitations and whether it actually answers the request.

## 6. Output structure conventions

- Use a memo format suitable for `gap-analysis-memo.docx`.
- Lead with an executive summary that includes concise diligence posture and an overall issue overview.
- Include a clearly labeled severity scale near the top and use it throughout.
- Organize the body by diligence category or request group, not as a flat list.
- Within each category, present issue-by-issue analysis in a consistent row-style format with the requested reference, current status, severity, gap characterization, significance, and follow-up.
- Include a short section for escalation items that may indicate regulatory non-compliance, approval risk, or chain-of-title / authority issues requiring immediate attention.
- End with a Recommended Actions section that uses imperative verbs, names the responsible role, and provides a timing anchor or urgency level.
- Keep the writing substantive and decision-useful; avoid merely restating the VDR index or tracker.
