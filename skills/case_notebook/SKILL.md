# 办案手记

## 概述

| 项目 | 内容 |
|------|------|
| **能力名称** | 办案手记 |
| **核心功能** | 生成并管理案件核心工作底稿，整合案件事实、法律问题、待办事项和庭前准备 |
| **适用领域** | 诉讼案件、仲裁案件、非诉项目全流程 |
| **输入材料** | 案件材料、法院文书、当事人信息、证据材料 |
| **输出格式** | 结构化Markdown底稿文档 |
| **关键原则** | 全面性（覆盖所有办案环节）、及时性（随案更新）、可追溯（每个结论标注来源） |

## 法律声明

> **重要提示：** 本技能生成的办案手记为内部工作底稿，不构成对外的法律意见。使用者应注意：第一，办案手记中的调研结论和判断意见为过程性记录，最终的法律意见应当经过审核。第二，手记中的案件事实来源于当事人提供和自行检索，未经法定质证程序确认。第三，办案手记涉及客户保密信息，应当严格保密。第四，手记中的待办事项应当及时跟进，避免遗漏。

## 一、核心概念

### 1.1 什么是办案手记

办案手记是每个案件办理过程中形成的内部工作底稿文件，核心作用是整合办案信息、资料，便于查看与避免遗漏。

**手记包含五部分：**

1. **案件事实**：经整理的大事记和事实梳理
2. **法律问题及调研结果**：涉及的法律问题和对应的调研结论
3. **待办事项及待收集资料**：需进一步查证的问题和待补充的资料
4. **庭前准备工作**：庭审策略、问题预判、证据准备
5. **办案心得**：过程中的思考和经验总结

### 1.2 为什么要形成办案手记

1. **避免重复劳动**：调研成果及时固化，同类问题不需重复研究
2. **经验积累**：如同"收纳盒"将散落的珍珠集中保存
3. **衍生实用文件**：从手记中提炼法规摘要、质证意见等
4. **保障问题落地**：实时记录待办事项，确保每个问题都被调研清楚
5. **知识沉淀**：案件办结后基于手记做知识整理和业务宣传

### 1.3 办案手记的基本结构

```
# 案件名称 — 办案手记

## 一、案件基本信息
案号、当事人、法院、承办团队、收案日期

## 二、案件事实（大事记）
按分类整理的时间线事实

## 三、法律问题及调研结果
| 问题 | 结论 | 法律依据 | 案例 | 备注 |

## 四、待办事项及待收集资料
- [ ] 待办1（负责人，截止日期）
- [ ] 待办2

## 五、庭前准备
- 争议焦点预判
- 模拟问题准备
- 证据清单

## 六、法规摘要
相关法律法规摘要

## 七、办案心得与思考
```

## 二、操作步骤

### 2.1 初始建立

收案后立即开始建立办案手记。初始内容包括案件基本信息和初步了解的事实。同步整理问题清单，记录需要调研的法律问题和需要向委托人确认的事实。

### 2.2 持续更新

伴随案件推进，手记需要不断更新：新获取的材料和事实及时补充；调研完成的问题及时记录结论；待办事项完成后标记为已完成；庭前准备内容逐步完善。在后续草拟分析意见或答辩状时，发现大事记中未记录的事实，同步补充。

### 2.3 衍生文件产出

基于办案手记，可以衍生出以下实用文件：法规摘要（提交法庭参考）、质证意见、代理词要点、结案报告、知识整理（一问一答式）、业务宣传文章。

## 三、质量检查清单

- [ ] 案件基本信息和当事人信息已记录
- [ ] 大事记已按分类和时间线梳理
- [ ] 所有法律问题已形成调研结论
- [ ] 待办事项列表已建立且定期更新
- [ ] 每项调研结论标注了法律依据和案例来源
- [ ] 庭前准备内容已完善
- [ ] 法规摘要已准备
- [ ] 办案心得已记录

## 四、关联技能

| 关联技能 | 关系 | 说明 |
|----------|------|------|
| 法律研究 | 并行 | 调研结论是手记的重要组成部分 |
| 时间线生成 | 上游 | 大事记是手记的事实基础 |
| 案件管理 | 并行 | 手记是案件管理的核心底稿 |
| 文书起草 | 下游 | 手记内容可衍生为正式文书 |
| 团队知识共享 | 下游 | 手记内容整理后可纳入知识库 |

## 五、局限性

第一，办案手记的效果依赖于及时更新，数据滞后可能导致遗漏。第二，手记中的法律判断是初步意见，正式法律意见书需另行出具。第三，手记内容涉及客户保密信息，应当严格管理使用权限。
