# 案件事实时间线生成

## 概述

| 项目 | 内容 |
|------|------|
| **能力名称** | 案件事实时间线生成 |
| **核心功能** | 从案件材料中自动抽取时间、事件、主体、金额等要素，按时间顺序生成结构化可视化时间线 |
| **适用领域** | 民事诉讼、刑事诉讼、行政诉讼、仲裁、合规调查、尽职调查 |
| **输入材料** | 起诉状、答辩状、证据材料、合同、聊天记录、邮件往来、会议纪要、新闻报道等 |
| **输出格式** | Markdown表格、Mermaid时间线图、结构化事件列表 |
| **关键原则** | 客观性（不添加未陈述的事实）、完整性（不遗漏关键时间节点）、可追溯性（每个事件标注证据来源） |

## 法律声明

> **重要提示：** 本技能生成的时间线是基于输入材料的结构化整理，不构成法律意见或事实认定。时间线的准确性依赖于输入材料的完整性和真实性。使用者应注意：
> 1. 时间线仅反映输入材料中记载的事件顺序，不代表法院或仲裁机构认定的事实
> 2. 时间线上的事件描述应标注对应的证据来源，便于后续质证和核实
> 3. 存在事实争议的时间节点应标注争议状态
> 4. 时间线的法律意义（如诉讼时效起算、除斥期间届满等）需要结合具体法律规范进行判断
> 5. 建议将时间线作为案件分析的辅助工具，而非最终的事实认定结论

## 一、核心概念体系

### 1.1 时间线的基本要素

每条时间线记录应包含以下核心字段：

| 字段 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| **时间节点** | 必填 | 事件发生的具体日期或时间段 | 2024年3月15日 |
| **事件描述** | 必填 | 客观、简洁地描述发生了什么 | 双方签订《货物买卖合同》 |
| **涉及主体** | 必填 | 参与该事件的主体名称 | 甲方（买方）、乙方（卖方） |
| **行为性质** | 推荐 | 该行为的法律性质分类 | 合同成立/履行/违约 |
| **涉及金额** | 如有 | 与事件相关的金额 | 合同总价500万元 |
| **证据来源** | 必填 | 该事实所依据的材料 | 合同第2页、转账记录 |
| **争议标记** | 如有 | 标注该事实是否存在争议 | 有争议/无争议/待核实 |
| **备注** | 可选 | 其他需要说明的信息 | 该日期为快递签收日 |

### 1.2 时间粒度选择

根据案件复杂程度和需求，选择不同的时间粒度：

| 粒度层级 | 适用场景 | 时间精度 | 示例 |
|----------|---------|---------|------|
| **宏观时间线** | 案件整体脉络、诉讼策略制定 | 年月级别 | 2024年1月签约→2024年6月违约→2024年9月起诉 |
| **中观时间线** | 争议焦点分析、庭审准备 | 日级别 | 2024年3月15日付首付款→2024年6月30日交房→2024年7月10日发现质量问题 |
| **微观时间线** | 证据审查、事实细节确认 | 时/分级 | 2024年3月15日14:00转账→14:30发确认邮件→15:00收到回复 |
| **复合时间线** | 复杂案件综合管理 | 混合 | 主时间线（日级别）+ 关键事件展开（时级别） |

### 1.3 时间线的法律意义

| 时间节点类型 | 法律意义 | 需关注的问题 |
|-------------|---------|------------|
| 合同成立时间 | 确定合同效力时点、管辖权依据 | 是否与实际签署时间一致？ |
| 履行期限 | 确定违约成立时间 | 是否有展期或变更？ |
| 损害发生时间 | 确定侵权成立时间、诉讼时效起算 | 是否连续发生？ |
| 知道或应当知道时间 | 诉讼时效起算、除斥期间起算 | 是否有证据证明知悉？ |
| 催告/通知时间 | 确定程序条件是否满足 | 是否有送达证明？ |
| 起诉/仲裁时间 | 诉讼时效中断、程序启动 | 是否在时效内？ |
| 裁判/判决时间 | 确定终局性、上诉期间起算 | 是否已生效？ |

## 二、操作步骤

### 2.1 第一步：材料接收与理解

1. **接收材料**：获取需要生成时间线的案件材料集合
2. **识别材料类型**：确定材料的性质（法律文书、证据材料、合同文本、沟通记录等）
3. **了解案件背景**：快速把握案件的基本事实：当事人、法律关系、争议焦点
4. **确认时间范围**：确定时间线的起止范围（如从合同磋商到起诉）
5. **识别关键节点**：在阅读材料过程中标注可能的关键时间节点

### 2.2 第二步：逐事件抽取与记录

对每份材料逐段分析，抽取时间相关事件：

**抽取规则：**

| 规则 | 说明 | 示例 |
|------|------|------|
| **显性时间抽取** | 直接提取明确的时间标注 | "2024年1月15日，甲方支付预付款" |
| **隐性时间推断** | 根据上下文推断时间 | "合同签订后第三天"→推算具体日期 |
| **相对时间标注** | 无法确定绝对时间时标注相对时间 | "合同签订后约一周"→标注"待确认" |
| **分段时间梳理** | 时间范围较长的标注起止 | "2024年3月至6月期间，双方持续协商" |
| **多版本时间标注** | 不同材料对同一事件时间记载不一致时 | 甲称3月15日、乙称3月16日→标注"争议" |

### 2.3 第三步：时间排序与分组

1. **排序**：将所有事件按时间顺序排列
2. **分组**：按自然阶段将时间线分段：
   - 合同磋商阶段
   - 合同签订阶段
   - 合同履行阶段
   - 违约/争议发生阶段
   - 争议解决阶段（协商、调解、仲裁/诉讼）
3. **合并**：同一时间点发生的多个事件合并展示
4. **去重**：多个材料描述同一事件时合并为一条

### 2.4 第四步：交叉验证

1. **时间矛盾检查**：检查不同材料中的时间记载是否存在矛盾
2. **证据对应**：每个事件标注对应的证据来源
3. **空白期识别**：标注时间线上信息缺失的时段
4. **关键节点重点审查**：对影响法律判断的关键时间节点进行重点核实

## 三、输出规范

### 3.1 通用规范

1. **格式统一**：时间格式统一使用"YYYY.MM.DD"或"YYYY年MM月DD日"
2. **主体标准化**：同一主体在不同材料中的表述统一为标准名称（首次出现时注明别名）
3. **语言客观**：使用中性语言描述事件，避免使用"故意""恶意""违约"等评价性词汇
4. **数据可追溯**：每个事件标注具体证据来源（材料名称、页码/段落）

### 3.2 特殊标记规则

| 标记 | 含义 | 使用场景 |
|------|------|---------|------|
| ⚠️ | 时间待确认 | 材料中时间模糊，需要进一步核实 |
| 🔴 | 事实争议 | 双方对事件描述或时间存在争议 |
| 🟡 | 间接推断 | 时间非直接记载，系根据上下文推断 |
| 🔵 | 程序节点 | 诉讼/仲裁程序中的时间节点 |
| ⭐ | 关键节点 | 对案件结果有重大影响的时间节点 |

## 四、常见场景处理

### 4.1 合同纠纷案件

重点关注：
- 合同签订时间、约定履行期限、实际履行时间
- 催告时间及方式（书面/口头）
- 违约行为的持续期间
- 解除通知的送达时间
- 诉讼时效起算点

### 4.2 侵权纠纷案件

重点关注：
- 侵权行为发生时间
- 损害结果发生时间
- 权利人知道或应当知道的时间
- 因果关系的时间链条
- 持续侵权中的分段时间

### 4.3 劳动争议案件

重点关注：
- 劳动关系建立时间
- 工资支付周期
- 解除/终止劳动关系时间
- 仲裁申请时间
- 劳动监察投诉时间

### 4.4 知识产权案件

重点关注：
- 权利取得时间（专利申请日、商标注册日）
- 侵权行为发生时间
- 权利人发现侵权时间
- 证据保全时间
- 临时禁令申请时间

### 4.5 刑事案件

重点关注：
- 犯罪预备时间
- 犯罪行为实施时间
- 犯罪结果发生时间
- 到案/拘留/逮捕时间
- 审判时间节点

### 4.6 婚姻家庭案件

重点关注：
- 结婚登记时间
- 感情破裂关键事件时间
- 分居起止时间
- 财产取得时间
- 子女出生时间
- 离婚起诉时间

## 五、实务经验与方法（团队实践精华）

### 5.1 大事记的分类整理法

大事记应当按照内容进行分类，而非仅仅是按时间流水排列：

**标准分类维度：**
```
大事记分类体系（参考）
├── 相关主体情况
├── 签约情况
├── 付款情况
├── 履行情况
├── 争议内容
├── 对方主要观点
├── 起诉/仲裁进展情况
└── 其他
```

**操作要点：**
- 每个分类项下的事实，按时间线梳理
- 先在整体上按分类归集，再在每个分类内按时间排列
- 分类与分类之间保持时间轴的平行对照关系

### 5.2 事实来源标注与区分

在记录事实时，必须区分"资料显示"和"委托人所述"：

| 标注方式 | 含义 | 使用场景 |
|----------|------|---------|
| **无标注** | 经文件/证据确认的事实 | 合同、邮件、聊天记录、银行流水等 |
| **据悉/据委托人介绍** | 委托人陈述但尚无文件证实 | 委托人描述但未提供文件 |
| **据资料显示** | 文件中有记载但尚未核实 | 初步阅读材料时的记录 |
| **待核实** | 存在矛盾需进一步确认 | 不同来源信息冲突 |

**关键原则：**
- 如果资料显示与委托人介绍的事实不符，应同时标注，并在大事记中注明不一致之处
- 后续核查清楚后，及时更新标注状态
- 这一标注方式既是工作方法，也是风险控制手段——避免将未经核实的事实当作确定事实使用

### 5.3 持续更新机制

大事记是动态文件，需要伴随案件进展不断更新：

1. **初始建立**：在第一次阅读完所有案件材料后，建立初步大事记框架
2. **随案更新**：每当获取新材料或发现新事实时，更新对应分类和新增条目
3. **交叉补充**：在后续草拟分析意见、答辩状等文书时，如果发现大事记中没有记录的事实，同步补充
4. **版本管理**：大事记文件命名应包含日期，保存历史版本，以免丢失进度

### 5.4 阅读材料的基本方法

在整理大事记之前，先按以下方法阅读案件材料：

1. **先搞清文件分类**：看清楚委托人一共提供了哪几大类的文件，建立整体认知
2. **优先看核心资料**：先看起诉状、证据目录、答辩状、委托人提供的情况说明、案情反映等核心资料
3. **快速归纳**：对每一份材料，先看主要内容，快速归纳其核心意思
4. **反复读核心证据**：对于合同、会议纪要等核心证据，需要反复、仔细阅读

## 六、质量检查清单

```markdown
□ 大事记是否按内容分类而不是单纯按时间排列？
□ 每个分类项下是否按时间线梳理？
□ 事实来源是否标注清楚（资料显示 vs 据悉）？
□ 资料与委托人陈述不一致处是否已备注？
□ 大事记是否随着新材料的获取持续更新？
□ 核心材料（合同、会议纪要）是否已反复阅读？
□ 案卷材料是否先进行了分类整理？
□ 每个事件是否标注了对应的证据来源？
□ 时间线上是否存在空白期需要补充？
□ 关键时间节点（诉讼时效、合同期限等）是否已特别标注？
```

| 关联技能 | 关系 | 说明 |

| 事实抽取 | 上游 | 时间线生成需要先进行事实抽取 |

| 诉辩策略分析 | 下游 | 时间线支撑诉讼策略制定 |
| 案件管理 | 并行 | 时间线是案件管理的重要工具 |

## 八、局限性

1. **时间精度限制**：对于时间模糊的材料，无法精确确定事件发生顺序
2. **材料依赖**：时间线的完整性完全依赖输入材料的完整性
3. **事实未经过认定**：时间线反映的是材料中记载的事件，不等于法院认定的事实
4. **复杂因果链**：对于多因一果、一因多果等复杂因果关系，简单时间线可能无法完全反映
5. **主观判断残余**："关键事件"的选择涉及专业判断，不同分析者可能有不同侧重
