# 会议记录

## 概述

| 项目 | 内容 |
|------|------|
| **能力名称** | 会议记录 |
| **核心功能** | 对法律会议、案件讨论会、案情沟通会进行结构化记录和待办事项追踪 |
| **适用领域** | 内部案件讨论、与委托人沟通、与对方协商、庭审后复盘 |
| **关键原则** | 及时性（会后尽快整理）、完整性（记录关键讨论和结论）、可追溯（待办事项有人负责） |

## 一、核心方法

### 1.1 会议记录的基本结构

```
# 会议记录 — [案件名称/主题]

## 基本信息
- 时间：xxxx年xx月xx日 xx:xx-xx:xx
- 地点/方式：xx会议室/电话/视频
- 参会人员：xxx、xxx
- 记录人：xxx

## 会议内容
### 第一部分：案件基本情况回顾（如有需要）
### 第二部分：讨论要点
1. 议题一：讨论内容/各方意见/结论
2. 议题二：讨论内容/各方意见/结论

## 达成的共识
1. xxx
2. xxx

## 待办事项
| 序号 | 事项 | 负责人 | 截止时间 | 状态 |
|------|------|--------|---------|------|
| 1 | xxx | xxx | xxx | 进行中 |

## 下次会议安排（如有）
- 时间：xxx
- 议题：xxx
```

### 1.2 会议记录要点

1. **分类记录**：将会议内容分为"基本情况"和"重点问题"两部分
2. **录音辅助**：重要会议进行录音，确保记录完整
3. **即时记录待办**：会议中确定的待办事项当场记录，会后第一时间发出
4. **材料跟进**：会议中确定需要补充的材料，会后尽快整理清单发给委托人

### 1.3 案件讨论会的正确方式

案件团队讨论不是"辩论会"而是"剥洋葱"：每个参与者从不同角度提出问题；层层深入分析问题；最终形成对案件事实和法律问题的全面认识。

## 二、操作步骤

### 2.1 会前准备

明确会议议题和参会人员；准备案件材料摘要；确定需要讨论的重点问题列表。

### 2.2 会中记录

记录讨论的要点和各方意见；特别记录达成的共识和分歧；记录待办事项和负责人；重要会议进行录音。

### 2.3 会后整理

会后24小时内完成记录整理；将待办事项清单发给相关人员；归档保存会议记录；重要结论更新到办案手记。

## 三、质量检查清单

- [ ] 会议基本信息（时间、地点、参会人）已记录
- [ ] 讨论要点已按议题分类记录
- [ ] 达成的共识已明确记录
- [ ] 待办事项有明确负责人和截止时间
- [ ] 会后24小时内已发出会议记录
- [ ] 待办事项已更新到办案手记
- [ ] 重要会议是否已录音（存档备查）

## 四、关联技能

| 关联技能 | 关系 | 说明 |
|----------|------|------|
| 办案手记 | 下游 | 会议结论更新到办案手记 |
| 客户沟通 | 并行 | 与委托人的会议记录是客户沟通的重要组成 |
| 案件管理 | 并行 | 会议待办事项是案件管理的一部分 |

## 五、局限性

第一，会议记录的完整性受限于记录速度和参与程度。第二，录音记录需要征得参会人同意。第三，口头讨论中的即时想法可能需要后续核实。

## Legalwork 执行增强

### 何时触发

当用户上传会议录音转写、聊天记录、案情沟通纪要、庭后复盘内容，或要求“整理会议纪要、生成待办、提炼共识/分歧、同步到案件记录”时，使用本 Skill。

### 软件联动

从 Legalwork 附件和工作区文件中抽取会议原文，输出会议纪要 Markdown；如用户需要，可同时生成待办表、客户补充材料清单、案件手记更新片段。对未确认的口头意见标注“待核实”，不要当作既定事实写入案件结论。
