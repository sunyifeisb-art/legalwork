# 案件管理

## 概述

| 项目 | 内容 |
|------|------|
| **能力名称** | 案件管理 |
| **核心功能** | 对诉讼和仲裁案件进行全流程管理，涵盖案件档案管理、进度跟踪、期限监控、团队协作和风险管理 |
| **适用领域** | 民事诉讼、刑事诉讼、行政诉讼、仲裁程序、劳动仲裁、行政复议、非诉项目 |
| **管理阶段** | 立案前评估、立案阶段、审理阶段、判决/裁决阶段、执行阶段、结案归档 |
| **核心维度** | 时间管理（期限监控）、文档管理（档案归档）、风险管理（案件评估）、团队管理（分工协作） |

## 法律声明

> **重要提示：** 本技能提供的案件管理辅助功能，不构成法律意见或案件处理建议。使用者应注意：第一，案件管理系统中的数据应当及时更新，确保信息的准确性和时效性。第二，系统中的期限监控功能仅为辅助提醒，不替代律师对法定期限的专业判断和核实。第三，案件档案中涉及客户保密信息的，应当严格遵守保密义务。第四，案件风险评估仅为初步判断，最终的案件策略应当由主办律师结合具体情况确定。第五，系统生成的案件报告和统计数据应当与实际情况核对后使用。

## 一、核心概念体系

### 1.1 案件管理的核心要素

案件管理涉及五个核心要素：第一是案件档案，包括案件基本信息、当事人信息、承办律师、受案法院、案号、案件类型等基础数据。第二是期限管理，包括诉讼时效、举证期限、上诉期限、申请执行期限、开庭日期等关键时间节点。第三是文档管理，包括起诉状、答辩状、证据材料、代理词、判决书等案件文书的归档和管理。第四是进度管理，记录案件从立案到结案的各阶段进展状态。第五是风险管理，对案件的可能结果、执行风险、收费风险等进行评估和监控。

### 1.2 案件管理的基本原则

案件管理应当遵循以下基本原则：第一，及时性原则，案件信息和进展应当及时更新，防止因信息滞后导致决策失误。第二，完整性原则，案件档案应当包含从收案到结案的全过程记录，做到每一个关键节点都有据可查。第三，保密性原则，案件信息涉及客户隐私和商业秘密，应当严格保密。第四，标准化原则，案件分类、编码、归档等应当采用统一标准。

### 1.3 案件分类体系

案件可按以下维度进行分类管理：按案件类型分为民事案件、刑事案件、行政案件、仲裁案件、劳动仲裁案件、非诉项目等。按案件阶段分为立案前评估阶段、立案阶段、审理阶段、判决/裁决阶段、执行阶段和结案归档阶段。按案件风险等级分为低风险案件、中风险案件、高风险案件和特高风险案件。按客户类型分为个人客户案件、企业客户案件、政府客户案件等。

## 二、操作步骤

### 2.1 第一步：收案登记

收案时应当记录以下基本信息：案件名称（简要概括案件核心内容）、案件编号（按事务所统一编码规则编制）、当事人信息（各方当事人名称/姓名、联系方式、送达地址）、承办律师团队（主办律师、协办律师、实习律师）、收案日期和来源渠道（客户直接委托、合作律师介绍、招投标等）、收费安排（固定收费、风险代理、计时收费等）、利益冲突检索结果（确认是否存在利益冲突）。

### 2.2 第二步：案件策划与分工

收案后应当制定案件处理计划，明确案件的核心目标、关键争议焦点、初步诉讼策略和时间安排。确定团队分工，明确各成员的工作职责和完成时限。建立案件工作文件体系，包括证据材料索引、法律法规汇编、类案检索报告等。

### 2.3 第三步：进度跟踪与期限监控

案件进入正式程序后，应当建立进度跟踪机制。记录每一个程序节点的时间、内容和结果，包括立案日期、开庭日期、证据交换日期、裁判日期等。设置期限预警机制，对诉讼时效、上诉期限、申请执行期限等关键时限提前设置提醒。开庭前应当进行模拟法庭或庭前会议，确保证据和代理意见准备充分。

### 2.4 第四步：文档管理

建立完整的案件文档管理体系。所有案件文书应当统一编号和归档，包括起诉状、答辩状、代理词、证据目录、质证意见、法律意见书等。对收到的对方文书和法院文书应当及时登记上传。建立文档版本管理机制，确保使用最新版本的文书。

### 2.5 第五步：结案归档

案件终结后应当及时进行结案处理。整理完整的案件档案，包括全部法律文书、证据材料、工作记录和往来函件。撰写结案报告，总结案件处理过程、裁判结果、经验教训和客户反馈。清理案件相关数据，完成收费结算。将案件档案移交档案管理部门归档。

## 四、实务经验与方法（团队实践精华）

### 4.1 核心卷宗"三大表"体系

每个案件都准备一本核心卷宗，包含三大表：

**第一大表：案件基本信息表**
```
案件名称：
案号：
受案法院/仲裁机构：
承办法官/仲裁员：
当事人信息（各方）：
承办律师团队：
收案日期：
收费安排：
重要日期（立案、开庭、判决等）：
案件状态：
```

**第二大表：期限监控表**
```
| 期限类型 | 起算日 | 截止日 | 剩余天数 | 状态 | 备注 |
|----------|--------|--------|---------|------|------|
| 举证期限 | 1月15日 | 2月15日 | 10天 | 进行中 | 待补充证据3份 |
| 上诉期限 | 3月1日 | 3月16日 | — | 已届满 | 未上诉 |
| 申请执行期限 | 4月1日 | 2年内 | — | — | 提醒客户 |
```

**第三大表：案件文档索引表**
```
| 序号 | 文件名称 | 版本 | 日期 | 来源 | 存放位置 |
|------|---------|------|------|------|---------|
| 1 | 起诉状 | v1.0 | 1月10日 | 原告 | 卷宗第1页 |
| 2 | 答辩状 | v2.0 | 1月28日 | 我方 | 卷宗第5页 |
```

**核心卷宗还应包含：**
- 起诉状及证据
- 答辩状及证据
- 质证意见
- 法规原文及法规摘要
- 法庭传票
- 案件程序时间表

### 4.2 倒排工期法

从最终时间节点倒推各项工作安排，这是律师高效处理多案件的核心方法：

**步骤：**
1. **确定最晚时间节点**：如开庭日期、上诉截止日、提交文书截止日
2. **倒推各项工作**：基于最晚节点确定各项工作必须完成的时间
3. **设定中间节点**：为大项工作设定分阶段完成时间
4. **打出富余量**：每个步骤预留20%-30%的缓冲时间

**案例：以开庭日期倒排**
```
开庭日：3月15日
├── 庭前准备完成：3月13日（2天富余）
│   ├── 模拟法庭/庭前会议：3月10日
│   │   └── 代理词定稿：3月8日
│   ├── 证据整理装订：3月5日
│   │   └── 质证意见完成：3月3日
│   └── 法规摘要更新：3月1日
├── 法律文书提交法院：2月20日（答辩状等）
└── 案件材料全部阅读完成：1月30日
```

### 4.3 项目管理者视角

做好案件项目管理者，需要具备以下能力：

1. **时间预估**：一项工作要经过哪几个步骤、每个步骤需要多少时间，应该心中有数，并打出富余量
2. **团队协同**：牵头律师要设定每个人的完成时间，过程中跟进进度，发现问题及时解决
3. **"扔猴子"理论**：接到工作后先想哪些可以分配给团队小伙伴，第一时间安排出去并设好时间点
4. **每日梳理**：每天早晨梳理手头案件，确定优先顺序，按二八法则分配精力

### 4.4 剥洋葱工作法

思考问题要层层深入、不断提升，不能思考得过浅：

**方法：**
1. **第一层**：了解问题的表面信息
2. **第二层**：分析问题的法律性质和法律关系
3. **第三层**：研究相关的法律规定和司法实践
4. **第四层**：结合案件事实进行综合分析和判断
5. **第五层**：形成结论并考虑可能的变数和应对方案

**示例**：调研再审案件管辖问题
```
第一层：再审应在哪级法院？
第二层：涉及四级法院审级职能定位改革
第三层：2021年试点文件→试点期限到2023年
第四层：2023年民诉法修订草案二审删除再审管辖条款→继续适用旧规定
第五层：旧规定下管辖明确后，还要确认去哪个巡回法庭
```

### 4.5 每句话都要有出处

"尽量做到每一句话都有出处、无字不无出处"——这是保证法律工作质量的核心原则：

1. **事实陈述有证据支持**：每个事实都要标注对应的证据来源
2. **法律引用有原文依据**：引用的法条要核对原文，注明具体条款
3. **案例引用有案号可查**：每个案例标注案号、裁判法院和裁判日期
4. **学理观点有出处可追**：引用学术观点要注明作者、文章名称和出处
5. **交叉验证**：对关键事实和法律适用进行多来源交叉验证

## 七、局限性

第一，案件管理系统的效果依赖于使用人员的主动维护和及时更新，数据滞后可能导致管理决策偏差。第二，期限监控仅为辅助工具，不能完全替代律师对法定期限的独立核实。第三，案件风险评估受限于信息完整性，信息不充分时评估结论可能不准确。第四，不同律师事务所的案件管理流程和标准存在差异，本框架需要根据实际情况进行调整。
